The distribute_nine experiment was conducted using the same technical setup as the previous experiments in order to ensure comparability across all I-RAVEN configurations. Training was performed for 300 epochs using the Adam optimizer with a learning rate of 0.005 and a weight decay of 0.01. The training batch size was set to 128, while evaluation used a batch size of 32. The experiment relied on the previously generated I-RAVEN dataset together with the prepared train, validation, and test index files. During execution, checkpoints, logs, summaries, and metadata were continuously stored in the dedicated run directory.

The experiment completed successfully without runtime instability or implementation-related failures. However, the final performance remained considerably lower than the results obtained for the center_single configuration. After 300 epochs, the model achieved a final training accuracy of approximately 73.4% and a final test accuracy of approximately 46.8%. Throughout the later epochs, both training and test accuracy fluctuated within a relatively narrow range without exhibiting meaningful long-term improvement, suggesting that the model had reached a performance plateau under the current configuration.

The distribute_nine configuration represents the most complex grid-based setting among the experiments conducted so far. In contrast to center_single, which contains only a single centrally positioned object, the distribute_nine setup corresponds to a 3×3 spatial grid with substantially more objects and relational dependencies. This significantly increases the combinatorial complexity of the symbolic reasoning task. The model must simultaneously process and relate a larger number of object attributes and transformations, making the reasoning problem considerably more difficult.

An important observation is that the overall behavior of the experiment was highly similar to the previous distribute_four run. Both experiments converged toward training accuracies around 72–73% and test accuracies around 45–47%, despite the increased spatial complexity of the distribute_nine configuration. This suggests that the reproduced implementation may currently struggle with multi-object grid reasoning in general rather than specifically failing on the largest configuration alone. At the same time, the experiments remained technically stable throughout all 300 epochs, indicating that the lower performance is unlikely to originate from optimization collapse, data corruption, or runtime instability.

The discrepancy between the near-perfect center_single performance and the substantially lower distribute_four and distribute_nine accuracies will require careful discussion in the final replication report. One possible explanation is that the reconstructed implementation environment still differs in subtle ways from the original framework used in the paper, particularly because unavailable jactorch components had to be manually replaced with simplified PyTorch implementations. Another contributing factor may be the use of a single random seed rather than the multi-seed protocol described in the original paper. Nevertheless, the experiment confirms that the adapted replication pipeline can reliably execute long training runs on multiple I-RAVEN configurations while preserving stable training dynamics and reproducible execution behavior.
