The distribute_four experiment was conducted using the same technical setup as the previous center_single replication in order to maintain comparability across configurations. Training was performed for 300 epochs using the Adam optimizer with a learning rate of 0.005 and a weight decay of 0.01. The training batch size was set to 128, while the evaluation batch size was set to 32. The experiment used the generated I-RAVEN dataset together with index files created specifically for the train, validation, and test splits. Training logs, checkpoints, metadata, and summaries were automatically stored in a dedicated run directory throughout execution.

The experiment completed successfully, but unlike the center_single configuration, the final performance was substantially lower. The model achieved a final training accuracy of approximately 73.4% and a final test accuracy of approximately 45.7% after 300 epochs. Throughout the later training stages, both the training and test accuracies fluctuated around relatively stable values without further substantial improvement, indicating that the model had likely reached its optimization limit under the current configuration.

Compared to the nearly perfect performance obtained on the center_single task, the distribute_four configuration appears significantly more challenging for the SCL architecture. This outcome is expected to some extent because the distribute_four setup corresponds to a 2×2 grid structure containing more objects and more complex relational dependencies than the single centered object arrangement used in the previous experiment. The model therefore has to reason about a larger combinatorial space of symbolic relationships, increasing task complexity considerably.

An important observation is that the model did not exhibit catastrophic divergence or unstable optimization behavior. Training progressed consistently across all 300 epochs, losses remained bounded, and both training and evaluation metrics evolved smoothly throughout the run. This indicates that the implementation itself remained technically stable and that the lower performance is most likely attributable to task difficulty or remaining architectural differences rather than implementation failure.

The experiment therefore provides additional evidence that the adapted replication pipeline functions correctly across multiple I-RAVEN configurations. At the same time, the substantial performance gap between center_single and distribute_four suggests that the symbolic reasoning challenges represented by multi-object spatial configurations are significantly harder for the reproduced model to solve. This discrepancy will become particularly important during comparison with the original paper results, since it may reveal differences between the reconstructed implementation environment and the original training setup, especially regarding framework behavior, initialization details, random seeds, or omitted jactorch functionality.
